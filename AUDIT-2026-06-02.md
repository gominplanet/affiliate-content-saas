### Commit 2 — Security P0s (`7abd853`)

8 distinct fixes, none breaking, file-scoped:

- **NEW `lib/ssrf-guard.ts`** — exports `assertPublicHttpUrl(url)` that rejects non-http(s) schemes, raw private/loopback/metadata IPs (RFC1918, 127/8, 169.254/16 incl. cloud metadata, 100.64/10 CGN, 224+, etc), and known metadata hostnames. Applied to `/api/wordpress/test` (was unauthenticated SSRF — attacker probes 169.254.169.254 for IAM creds, internal services, etc).
- **Facebook OAuth CSRF** — `/api/auth/facebook` start route now sets `state=user.id`; callback rejects if state missing or ≠ session user. Was open to victim-binding-attacker's-Page attack.
- **Twitter OAuth defense-in-depth** — already passed state but never verified it matched session user. Closed (so if anyone ever swaps for `createAdminClient` later, it's still safe).
- **`/api/wordpress/debug-state` cross-tenant info leak** — was returning EVERY connected WP hostname + user_id prefix across the platform to any logged-in caller. Scoped to the caller's own row.
- **Stripe webhook race condition** — `customer.subscription.deleted` matched on customer_id alone. A churned-then-resubscribed user could get a stale .deleted event flip their NEW subscription to trial. Now also requires `stripe_subscription_id` match.
- **Stripe webhook metadata trust** — `checkout.session.completed` trusted `metadata.tier` without checking the actual `priceId`. Now derives tier from `line_items[].price.id` via `PRICE_TO_TIER`.
- **`.single()` → `.maybeSingle()` crash bugs** in 7 routes (profile, admin/announcement, admin/user-lookup, auth/gsc/callback, auth/youtube/disconnect, photobooth, wordpress/setup-site, blog/refresh-images). Each was reading an OPTIONAL row but crashing the route with `.single()` when the row didn't exist (fresh trial users).
- **Admin user-lookup pagination** — was `perPage: 1000` once, silently 404'd for users on page 2+. Now walks pages until found, capped at 50k.

### Commit 3 — Performance wins (`b1e7d54`)

Real impact, real measurements:

- **Dashboard `youtube_videos` count** — was `.select('id')` + `videos.length` (full row fetch, transcript rows are 5-50KB each). Now `count: 'estimated', head: true`. Saves 5-50KB row payload + ~100-500ms TTFB.
- **Dashboard 4× `count: 'exact'`** → `'estimated'`. ai_usage scales fast (≥3 rows/generation). Exact = full table scan; estimated = pg_stat read. ~800ms → ~300ms once ai_usage crosses ~50K rows.
- **`blog/generate` parallelized** — 3 sequential awaits (video/brand/integrations) → `Promise.all`. **600ms → 200ms** before the AI call starts.
- **`blog/generate` deduped** — the YouTube-token fallback was re-fetching `integrations` (~80ms saved per gen).
- **`seo/overview` per-site loop** — was serial (each ~800ms). Multi-site Pro users with 3 sites paid 2.4s waterfall. **Parallelized → ~800ms**.
- **`seo/overview` post_seo SELECT** — was `*` (pulled 10KB JSONB `score_detail` per row × 200 rows = 2MB). Narrowed to 10 scalar fields actually rendered. **Payload -60%, parse -300ms**.
- **`getPublishedPostIds()`** — was a serial 10-page loop (5s on 600-post sites). Rewritten: page 1 first to discover `X-WP-TotalPages`, then pages 2..N via `Promise.allSettled`. **5s → ~700ms**.
- **`cron/process-scheduled`** — published rows serially (25 × 1-4s = 100s, past `maxDuration=60` → silent tail drop). Now `Promise.allSettled` over all rows. **Capacity 12 → 25+ per tick**.

### Commit 4 — Frontend P0s (`2ef625d`)

- **`components/TutorialVideo.tsx`** — fixed React Hooks rule violation (early `return null` BEFORE `useState`/`useEffect`). Latent crash any time `resolvedVideoId` transitioned between renders. Hooks now run unconditionally.
- **`components/layout/Sidebar.tsx`** — replaced 2 holdout `alert()` calls in cache-purge handler with `toast.error` / `toast.success`. Matches the rest of the app (task #110 sweep missed these two).

## What DIDN'T ship (needs your input)

These are real findings the audit surfaced, but each touches enough that I want explicit sign-off rather than landing them while you sleep:

### Security (defer)

- **WordPress App Passwords + social tokens stored PLAINTEXT** in `integrations`. If RLS bypass or service-role key leaks, every connected site's admin creds are exposed. Fix needs a schema migration + key-management decision (pgsodium / Vault / app-level symmetric).
- **Newsletter signup HMAC milestone** — `/api/newsletter/subscribe` already has a `TODO milestone 1.5: HMAC the form payload` comment. Currently an attacker can submit ANY creator's user_id and flood-spam confirmations from the creator's Resend sender → kills deliverability. Real attack surface.
- **`app/api/proxy-short/[videoId]`** is unauthenticated and proxies Storage URLs. A botnet with known UUIDs can burn your Vercel bandwidth quota. Needs rate-limiting decision.
- **Stripe webhook idempotency** — no `processed_event_ids` table. Stripe retries every webhook on 5xx, and you can replay manually. With `.deleted` followed by an out-of-order stale `.updated`, a tier could flip back. Worth an event-id dedup table.

### Performance (defer)

- **Public PNG bloat** — `automation-hub.png` (2MB, 1376×768), `mvp-affiliate-logo.png` (876KB, 1254×1254). Landing LCP -1.5s on slow 3G if converted to WebP/AVIF + resized. Touches design QA.
- **`/content` page full-row select** of `youtube_videos` — transcript column makes each row 10-50KB. For 600-video creators, that's 15MB per page load. Need UI side re-check first to verify nothing reads `transcript`.
- **7 routes still doing `select('*')` on `integrations`** (blog/twitter-post, blog/threads-post, blog/pinterest-preview, face-models, brand page, etc). Each pulls 30+ columns. Touch too many callers to sweep tonight.
- **Edge runtime candidates** — `/api/wp-version`, `/api/proxy-image`, `/api/announcement`, `/api/newsletter/unsubscribe`. Pure I/O, would shave 200-400ms cold-start. Each needs a node-API audit first.
- **`/api/newsletter/send`** runs Resend in chunks of 50 in parallel. On a Pro 5k-list, that's a 2-min request — should be deferred to a background job (newsletter_jobs table + cron worker).

### Frontend (defer)

- **15 `window.confirm()` calls** for destructive actions across studio/photobooth/collaborations/content/script/campaigns/newsletter (incl. newsletter SEND confirmation — the highest-blast-radius action in the app). Needs a `<ConfirmDialog>` component or sonner-style action toast pattern.
- **All `<label>` elements lack `htmlFor`/`id` pairings** across auth + setup. Screen readers don't link labels to inputs; click-to-focus also broken. Sweeping change, ~50 forms.
- **~12 modals missing focus traps** — no `<dialog>`, no `role="dialog"`, no `aria-modal`, no Escape-to-close, no scroll lock. Needs a shared Modal primitive (or Radix Dialog).
- **Loading state holes** on SEO, newsletter, content list, comparison, billing — show "Loading…" with no shape for 200-2000ms.
- **No 401 / session-expired interceptor** — every page silently fails after token expiry. Needs an `apiFetch()` wrapper.
- **33 `console.log/warn`** in blog/generate + youtube/generate-thumbnail leaking user payload data to Vercel logs. Needs a `DEBUG_BLOG_GEN` env flag.

### Code hygiene (defer)

- **~40 `as any` casts** in twitter-post, bluesky-post, linkedin-post, telegram-post, facebook-post routes — same row-shape laundering pattern. Extract a typed `loadBlogPostContext(postId)` helper to replace all of them.
- **9 `fal.subscribe(... as any, ...)` + `(result.data as any)?.images`** in youtube/generate-thumbnail. Wrap `@fal-ai/client` types once.
