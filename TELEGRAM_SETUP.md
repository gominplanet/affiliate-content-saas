# Telegram fan-out setup

Telegram auto-publish is a Pro plan feature. Reviews fan out to each
user's own Telegram channel via the Bot API.

Architecture:
- **One shared bot** for the whole MVP Affiliate app (token in
  `TELEGRAM_BOT_TOKEN` env). Created once via `@BotFather`.
- **Each user adds the shared bot** as an admin to their own channel,
  then pastes the channel ID into MVP's Integrations page.

---

## One-time platform setup (you do this once)

### 1. Create the MVP Affiliate bot

1. Open Telegram → search for `@BotFather` → start a chat.
2. Send `/newbot`.
3. Name: **`MVP Affiliate`**
4. Username: **`MVPAffiliateBot`** (or any available `*_bot` / `*Bot` handle).
5. BotFather replies with a token that looks like
   `1234567890:AAH-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`. Copy it.
6. Customize the bot a bit while you're there:
   - `/setdescription` — short description, e.g.
     *"Posts your MVP Affiliate reviews to your Telegram channel automatically. Add me as an admin to your channel and paste the channel ID in your MVP Affiliate dashboard."*
   - `/setuserpic` — upload a logo image so the bot avatar matches your brand
   - `/setjoingroups` — Disable (we don't need it joining groups)
   - `/setprivacy` — Enable (more privacy-friendly default for channel admin)

### 2. Add the token to env vars

**Vercel** → Settings → Environment Variables → Add:
- Key: `TELEGRAM_BOT_TOKEN`
- Value: paste the BotFather token
- Environments: Production + Preview
- Sensitive: **ON** (this is a real credential)

**Local `.env.local`** for testing:
```
TELEGRAM_BOT_TOKEN=1234567890:AAH-your-bot-token-here
```

Redeploy Vercel so the env var bakes in.

### 3. Run the migration

```bash
# In Supabase dashboard → SQL Editor, paste the contents of:
supabase/migrations/014_telegram_integration.sql
```

This adds:
- `integrations.telegram_channel_id` (text) — user's channel ID
- `integrations.telegram_channel_title` (text) — cached display name
- `blog_posts.telegram_message_id` (text) — tracks which posts have been pushed

## Per-user setup (what your customers do)

The Integrations card in `/setup?tab=integrations` walks them through it,
but here's the gist:

1. Create or open the Telegram channel they want reviews posted to.
2. Tap the channel name → **Administrators** → **Add Administrator**.
3. Search **`@MVPAffiliateBot`** (or whatever you named yours). Add it
   with **Post Messages** permission. The bot must be an admin — adding
   it as a regular member is not enough.
4. Make the channel **public** with a username (`@somethingreviews`) —
   simplest. Or grab the numeric channel ID from
   [`@getidsbot`](https://t.me/getidsbot) for private channels.
5. Paste the ID into MVP → Integrations → Telegram → Connect.

MVP calls Telegram's `getChat` to verify the bot can actually reach the
channel before saving, so users get an immediate "✅ Connected to
*MyReviews*" or a clear error if the bot isn't admin yet.

## How posts look

For each published review:
- **With a featured image** (most common): posts via `sendPhoto` — the
  review thumbnail appears as the lead visual, with a 3–5 line caption
  and a "Read the full review →" link.
- **Without an image**: falls back to `sendMessage` with link preview
  enabled.

Caption is generated by Claude Haiku from the blog title + excerpt,
matching the user's Brand Profile voice. Same agent pattern as the
Bluesky/Threads/LinkedIn fan-out routes.

## Troubleshooting

- **"Couldn't reach that channel" on Connect** → user hasn't added the
  bot as admin yet, or the channel ID is wrong. Have them double-check
  the channel's admin list shows the bot, and re-paste the ID.
- **Bot can read but can't post** → admin permissions were granted
  without "Post Messages". Have user re-edit admin permissions.
- **Posts arrive but caption looks broken (random `\` escapes)** →
  Telegram MarkdownV2 escaping bug. Re-check `services/telegram.ts`
  `escapeMarkdownV2()` — make sure all reserved chars are escaped
  before the caption is assembled.
